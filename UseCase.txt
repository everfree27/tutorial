Use case briefly
각각의 유즈케이스가 뭘하는지 정확하게 정리

용어 정리 - Administrator / Gym

기어S2 와 웹과 연동 가능한지
연동이 된다면 부트스트랩
안된다면 안드로이드


1. Check attendance - 
지하철처럼 칸막이가 있어서 출입시에 열림과 닫힘이 되고 출입자 인증
해당 출입자 같은 기어 S2로 짧은 시간 안에 재 출입 금지
시스템에 로그 기록


2. Join gym - 
트레이너 개인정보(성별,나이,경력,핸드폰번호,사진)
트레이니 개인정보(성별,나이,몸무게,키,질병여부,사진,핸드폰번호,등록일,만료일,PT등록여부)
기어S2에 사용자에 맞는 식별가능한 정보 입력
(시스템과 기어S2에 아이디 번호 부여)
트레이니가 원하는 트레이너의 정보를 (체육관 컴퓨터(보류))로 확인하고 매칭한다


3. Add work out info - 
트레이니가 기구에 부착된 NFC 를 작동해서 현재 사용중이라는 것을 다른 트레이니에게 알린다.
트레이니가 스케쥴에 적힌 운동을 수행한다.
하루 데이터를 트레이니가 트레이너에게 보낸다.


4. Manage trainee - 
트레이너가 트레이니 정보 받는다
트레이너만 접근가능한 스케쥴러(운동목록,운동량,운동시간...) 제작
완성된 스케쥴이 시스템에 저장  

 
5. add diet info -
트레이니의 식단을 작성한다

6. manage user - 
시스템에 사용자를 등록,삭제,조회한다.
